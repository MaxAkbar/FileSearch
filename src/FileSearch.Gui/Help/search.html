Search Workflow

Run a focused search.

Use the Main tab to choose whether you are searching contents, file names, folder names, or both, then set the folder and recursion scope. Use Options and Dates only when you need tighter control.

Main Tab

Field What it does Example
Containing text The text, name fragment, regex, or Boolean expression to find. Recent searches are saved in the dropdown. error AND timeout
Search Chooses the target: file contents, file names, folder names, or file and folder names. File and folder names
File name pattern Optional semicolon- or comma-separated glob filter for files. Blank means all files for name searches or all supported files for content searches. *.cs;*.xaml;*.md
Look in The folder to search. Recent folders are saved in the dropdown. C:\Source\MyApp
Include subfolders Searches the selected folder and all child folders when enabled. On for solution-wide search.

Search Controls

  • Start runs the search. Pressing Enter in the Containing text field also starts it.
  • Cancel stops the current search and keeps any results already found.
  • The status bar shows current progress, skipped files, failed files, total hits, and elapsed time.

File Name Patterns

Patterns are filters, not the search text. They are matched against file names, not the full folder path. Use semicolons or commas to separate multiple patterns.

*.cs
*.cs;*.xaml;*.json
*.md,*.txt
To search for a file or folder by name, change Search to File names, Folder names, or File and folder names. To limit which files are considered, use File name pattern.

Scanned or Image PDF Search

Searching *.pdf only limits the search to PDF files. It does not turn on OCR by itself. For scanned or image-only PDFs, use these settings:

  • Set Search to Contents.
  • Set File name pattern to *.pdf.
  • Keep Office/PDF enabled.
  • Enable Image OCR.
  • If Use index is enabled, make sure the indexed location was built with Image OCR enabled; otherwise use Index > Manage indexed locations... > Rebuild selected.

The Sidebar

The sidebar on the left keeps frequently used scopes, folders, and searches one click away. Each section has its own search box, shows page controls when there are many matches, collapses independently, and the panel scrolls as a whole when the expanded sections are taller than the window.

Section What it does
Scopes One-click file name patterns. Presets cover all files, source and config, documents, and PDFs. New scope saves your own named pattern, such as *.cs;*.xaml;*.md; clicking a scope fills File name pattern.
Recent folders Folders you searched recently. Click one to set Look in. The X on a row removes that entry.
Indexed locations Folders covered by background indexing, with status badges and health details. Manage locations opens the management window. See Indexed Search.
Saved searches Your most recent full searches. Click one to restore the query, folder, patterns, search mode, and filters.

Search History

FileSearch keeps the most recent full searches and folders so repeated searches are quick. A search attempt is added to history when it starts, even if it is canceled or fails. History appears in the sidebar's Recent folders and Saved searches sections.

Quick Search

Use Quick Search for launcher-style searches from anywhere. It uses one search box, supports current folder, selected indexed locations, all indexed locations, and bounded machine metadata scopes, then closes after you act on a result.
