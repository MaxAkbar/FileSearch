Release Executables

Use the commands included with the installed app.

The release package installs the FileSearch desktop app, command-line app, and helper executables in the same folder. Use the desktop app for interactive work, the CLI for scripts, and let FileSearch manage the indexer and extractor helpers.

Desktop App

Launch FileSearch from the Start menu, Explorer integration, or by running the desktop executable from the install or portable folder.

.\FileSearch.Gui.exe
.\FileSearch.Gui.exe "C:\Projects"
.\FileSearch.Gui.exe --background
Command Behavior
.\FileSearch.Gui.exe Opens the main FileSearch window.
.\FileSearch.Gui.exe "C:\Folder" Opens FileSearch with Look in set to the folder when the folder exists.
.\FileSearch.Gui.exe --background Starts FileSearch without showing the main window. When background indexing at sign-in is enabled, FileSearch starts the indexer and exits.

If FileSearch is already running, launching the desktop executable activates the existing instance and applies any folder argument.

Command-Line App

Run the CLI executable from the install or portable folder. With no subcommand it opens the interactive REPL. In the REPL, type help for commands, options for current settings, or type a query directly to search the current folder.

.\FileSearch.Cli.exe
.\FileSearch.Cli.exe search "connection string" --path C:\Projects --mode plain --json
.\FileSearch.Cli.exe index locations --json
.\FileSearch.Cli.exe workflow list
.\FileSearch.Cli.exe indexer status
Subcommand Executable usage
search .\FileSearch.Cli.exe search QUERY [--path FOLDER] [--mode plain|regex|boolean|unified] [--target contents|files|folders|names] [--ocr] [--no-docs] [--known-only] [--json|--jsonl|--csv|--markdown] [--output PATH] [--limit N]
index .\FileSearch.Cli.exe index build|rebuild|clear [FOLDER] or .\FileSearch.Cli.exe index locations|stats [FOLDER]|failures [--json|--jsonl|--csv|--markdown] [--output PATH]
workflow .\FileSearch.Cli.exe workflow list|folder|show NAME|validate NAME|run NAME [--dry-run] [--yes] [--no-hits]
indexer .\FileSearch.Cli.exe indexer status|start|pause|resume|shutdown|compact|refresh [FOLDER]|validate [FOLDER]|semantic [FOLDER]

Use --help after a subcommand, such as .\FileSearch.Cli.exe search --help, to print that subcommand's usage.

Release Executable Set

These executables live together in the install or portable folder.

Executable Purpose User action
FileSearch.Gui.exe Runs the desktop app, Quick Search window, settings, workflows, previews, shell integration, and indexed-location management. Launch from Start, Explorer integration, a shortcut, or directly from the install folder.
FileSearch.Cli.exe Runs interactive and one-shot search, index, workflow, and indexer commands from a terminal. Run directly from PowerShell, Windows Terminal, Command Prompt, scripts, or scheduled tasks.
FileSearch.Indexer.exe Keeps configured indexed locations current and exposes status, pause, resume, refresh, validation, and compaction operations to the desktop app. Manage it from Index, File > Settings, or the tray icon instead of launching it manually.
FileSearch.ExtractorHost.exe Runs document text extraction out of process so expensive or fragile extractors do not block the main app. No direct action is needed; FileSearch starts it when document extraction requires it.

Automation Boundary

The installed release is centered on the desktop app, Explorer integration, Quick Search, workflows, indexed locations, the CLI executable, and tray-managed background indexing. The bundled help documents the executables and arguments that ship with the installed app.

For repeatable user workflows, save a workflow in the desktop app and run it from the Workflows window or .\FileSearch.Cli.exe workflow run. For unattended scripting, use only executables and arguments present in the release package you are deploying.
