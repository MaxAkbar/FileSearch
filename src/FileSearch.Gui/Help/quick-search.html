Quick Search

Search from anywhere.

Quick Search is a small launcher-style window for everyday lookups. It opens over your current work, focuses the search box, streams results as you type, and closes after you act on a result.

Open Quick Search

FileSearch must be running for the global hotkey to work. If Keep index updated after closing window is enabled, closing the main window keeps FileSearch in the tray and the hotkey remains available. The default hotkey is Win+Shift+F.

Hotkey Notes
Win+Shift+F Default Quick Search shortcut.
Alt+Space Optional shortcut. Windows or another app may already use it.
Ctrl+Space Optional shortcut. Some editors and input tools may already use it.

Change the hotkey in File > Settings > Quick Search. If a shortcut is already reserved by Windows or another app, FileSearch reports that it could not register the hotkey. You can also open Quick Search from the FileSearch tray menu.

Search Names or Content

Quick Search always starts with fast filename and path matches. Turn on Content in the quick window to add indexed file-content matches after those instant results.

  • Content on searches file names, paths, and indexed text for the selected scope.
  • Content off searches only file names and paths, which is useful for launcher-style lookup.
  • The Entire machine metadata scope is metadata-only, so content search is available on indexed scopes.

Choose a Scope

The scope dropdown controls where Quick Search looks. Use indexed scopes for the fastest daily-driver searches.

Scope What it searches
Selected folder The folder shown in the Quick Search folder field. Use the folder button to change it before searching.
Selected indexed locations The indexed roots checked in File > Settings > Quick Search. If none are selected, all configured indexed locations are used.
All indexed locations Every indexed root configured in Index > Manage indexed locations.
Entire machine metadata Filename and path matches across ready fixed drives. This is a bounded metadata search, not a full content scan.

Use the Results

Results appear in stages. Filename and path matches are shown first; indexed content matches are added as the index query returns. The quick window keeps the list bounded so broad searches stay responsive.

  • Down moves from the search box into the results list.
  • Enter opens the selected result.
  • Esc closes the quick window.
  • Ctrl+R reveals the selected result in Explorer.
  • Ctrl+C copies the selected result path when the search box is not focused.
  • Ctrl+P pins the selected result.
  • F4 or Ctrl+I loads an inline preview.

Actions

Each result row has buttons for common actions:

  • Open opens the file in its default app and closes Quick Search.
  • Reveal opens Explorer with the file selected and closes Quick Search.
  • Copy path copies the full path and closes Quick Search.
  • Pin saves the result to the top of the quick window's pinned list and closes Quick Search.
  • Dragging a result out of the window starts a standard Windows file drag-and-drop operation.

Pinned Results and Preview

When the search box is empty, Quick Search shows pinned results. Pin frequently used files if you want them available before typing a query.

Preview is loaded on demand so opening the quick window stays fast. For content hits, preview shows matching lines with nearby context. For filename-only matches, preview may be empty if no content hit lines are available.

Use the full FileSearch window for deep search setup, index management, detailed previews, workflows, and reports. Quick Search is optimized for fast lookup and action.
