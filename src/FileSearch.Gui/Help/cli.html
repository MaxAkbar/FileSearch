Command Line

Automate FileSearch from a terminal.

The release package includes FileSearch.Cli.exe next to the desktop app. Use it for scripted searches, structured output, index maintenance, workflow runs, and background indexer control.

Open the REPL

Run the CLI from the installed or portable release folder without a subcommand to open the interactive prompt. Type help for commands, options for current settings, or type a query directly to search the current folder.

.\FileSearch.Cli.exe

Interactive Commands

Command Behavior
path C:\srcSets the current search root.
mode plain|regex|boolean|unifiedChooses the query parser.
target contents|files|folders|namesSearches content, file names, folder names, or both file and folder names.
case on|offToggles case-sensitive matching.
recursive on|offIncludes or skips subfolders.
docs on|offIncludes or skips Office/PDF/document extractors.
ocr on|offIncludes or skips image OCR and OCR fallbacks for scanned or embedded images.
known-only on|offRestricts content searches to known extractor extensions.
include, exclude, ext, exclude-ext, plain-extControls glob, extension, and custom text-extension filters.
index on|offAllows covered searches to use the local index.

One-Shot Search

Use search for automation. Results go to standard output unless --output is provided.

.\FileSearch.Cli.exe search "connection string" --path C:\src --mode plain --json
.\FileSearch.Cli.exe search invoice --path C:\docs --target files --json
.\FileSearch.Cli.exe search "serial number" --path C:\scans --ocr --markdown

Search output formats include JSON, JSON Lines, CSV, and Markdown. Use .\FileSearch.Cli.exe search --help to print the executable's current search usage.

Index Commands

The CLI can build, rebuild, clear, inspect, and report on the CSharpDB index without opening the desktop app.

.\FileSearch.Cli.exe index build C:\src --json
.\FileSearch.Cli.exe index locations --markdown
.\FileSearch.Cli.exe index failures --jsonl --output .\index-failures.jsonl

Use .\FileSearch.Cli.exe index --help to print the executable's current index usage.

Workflow Commands

Saved workflow JSON files can be listed, inspected, validated, and run from the CLI. Side-effecting workflow steps ask for confirmation in an interactive terminal; pass --yes only for unattended runs you have reviewed.

.\FileSearch.Cli.exe workflow list
.\FileSearch.Cli.exe workflow validate "Find TODOs"
.\FileSearch.Cli.exe workflow run "Find TODOs" --dry-run
.\FileSearch.Cli.exe workflow run "Find TODOs" --yes --no-hits

Background Indexer Commands

indexer commands use the same per-user named-pipe API as the GUI and tray indexer. Use them to check status, start or stop the worker, pause or resume indexing, queue root refreshes, validate index health, and compact the database.

.\FileSearch.Cli.exe indexer start
.\FileSearch.Cli.exe indexer status
.\FileSearch.Cli.exe indexer pause
.\FileSearch.Cli.exe indexer refresh C:\src
.\FileSearch.Cli.exe indexer validate C:\src
The CLI ships as FileSearch.Cli.exe in the release package. The examples on this page assume you are running from the installed or portable app folder.
