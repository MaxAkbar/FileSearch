Troubleshooting

Fix common search issues.

Most issues come from scope, query mode, file type filters, or Windows file permissions. The status bar usually tells you where to start.

No Results

  • Confirm the Look in folder exists and contains the files you expect.
  • Turn on Include subfolders if the files are below the selected folder.
  • Clear File name pattern or broaden it to include the target file types.
  • Check whether Skip unknown file types or Office & PDF documents is excluding files.
  • Switch to Plain text mode if Boolean operators or regex syntax are not intended.

Invalid Query

If the status bar reports an invalid query, review the selected search mode.

  • Boolean mode requires balanced quotes and parentheses.
  • Regex mode requires a valid .NET regular expression.
  • Plain text mode is the safest choice for literal text containing punctuation.

Search Is Slow

  • Add a file name pattern, such as *.cs;*.xaml, to avoid scanning unrelated files.
  • Disable Office and PDF extraction if you only need text/source files.
  • Enable Skip unknown file types when scanning folders with many binaries.
  • Add date or size filters to avoid very old, very small, or very large files.
  • Cancel the current search if the scope is too broad, then start again with tighter filters.
  • Index folders you search repeatedly and enable Use index. See Indexed Search.

Index Is Not Being Used

  • Confirm Use index is enabled next to the search controls or in the Manage indexed locations window.
  • The Look in folder must be inside an indexed location. Check the sidebar's Indexed locations section.
  • A location indexed without subfolders does not cover a recursive search below it. Check the location's Coverage in Manage indexed locations.
  • The search options must match what the location was indexed with (see Indexed with in the location details). After changing extraction options, use Rebuild selected.
  • When a search is not covered, FileSearch falls back to a live scan automatically — results are still correct, just slower.

Indexed Results Look Stale

  • Check the location's watcher badge: Watch disabled means file changes are not applied automatically.
  • Check whether background indexing is paused — the Index menu and the status bar show indexing activity.
  • Use Rebuild selected in Manage indexed locations to re-scan the location from scratch.

Preview Fails or Looks Incomplete

The Hits preview loads the selected file after the search. It can fail if the file was moved, deleted, locked, or cannot be decoded as text. For generated document formats, extracted text may not have the same layout as the original document.

Windows Integration Does Not Appear

  • Use File > Install Windows integration again and watch the status bar for errors.
  • Restart File Explorer if the context menu does not refresh immediately.
  • Use File > Remove Windows integration before reinstalling if the executable moved.

Store or MSIX Install Notes

For local sideload testing, the MSIX must be signed with a certificate trusted by the test machine. Microsoft Store submission uses the generated .msixupload package with the app identity and publisher reserved in Partner Center.
