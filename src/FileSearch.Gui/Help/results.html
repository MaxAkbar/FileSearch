Results

Review, refine, and open matching files.

Search results are grouped by matched file or folder. Select a result to see matching lines or name-match details, use the Filter tab to narrow the current result set, or open items directly from the grid.

Results Grid

Column Description
Name The matched file or folder name.
Hits The number of matching lines, spans, or name matches found for that result.
First match A short preview of the first matching line.
Location The folder containing the file.

Hits Tab

Selecting a result automatically opens the Hits tab. Content searches show matching line numbers and nearby context. File-name and folder-name searches show stored name-match details. For large or binary-backed document formats, the preview may be simplified depending on the extractor. OCR image hits include the recognized line's image-region label and can show standalone images with the OCR region highlighted. OCR PDF hits include the page and region label and can render the page with the OCR region highlighted. Embedded-image OCR hits from documents, emails, EPUBs, and archives include the member path and OCR-region label; when the embedded image can be read from the parent file, preview extracts it and highlights the OCR region.

Use Open OCR preview on OCR-backed results to open a dedicated FileSearch preview window at the exact highlighted region. The normal Open action now tries the best available source target first: OCR region preview for OCR hits, a PDF page link for page-anchored PDF hits, or a line-and-column editor target when Visual Studio Code is installed. If no exact target is available, FileSearch opens the parent file in its default app.

Summary Tab

The Summary tab shows search statistics: files matched, total hits, and elapsed time. The status bar gives additional live progress while a scan is running.

Filter Tab

The Filter tab narrows the current results in memory. It matches against file names and hit lines, so it is useful after a broad search.

  • Filtering does not re-open or rescan files.
  • Clear the filter field to show all results from the search again.
  • Run a new search when you need to change folder, file pattern, query mode, or date and size filters.

Sorting, Grouping, and Facets

The result toolbar can reorganize the current result set without scanning files again.

  • Sort orders visible results by relevance, recency, filename, or hit count.
  • Group keeps result cards together by file, folder, file type, or modified date.
  • Type, Folder, Date, Source, and Size facets narrow the visible result cards. Source shows whether the result came from the index or a live scan.
  • The clear button beside the facet controls resets all result facets.

Opening and Copying

Double-click a result row to open the file or folder. Right-click a row for more actions:

  • Open opens the exact source location when FileSearch can represent one, otherwise opens the file with the default Windows application or opens the folder in Explorer.
  • Open OCR preview opens FileSearch's exact highlighted OCR preview when the result has an image, scanned-PDF-page, or embedded-image OCR region.
  • Reveal in Explorer opens the containing folder and selects the file, or opens the folder itself.
  • Rename changes a file name in the same folder. Folder rename is not enabled from search results.
  • Move to Recycle Bin asks for confirmation before deleting a file and uses the Windows Recycle Bin.
  • Copy file path copies the full result path to the clipboard.
  • Copy folder path copies the containing folder path.
  • Add favorite saves the file in the Favorites sidebar section.
  • Pin result keeps the file available in Quick Search when no query is typed.
  • Drag a result card to another app or folder to start a Windows file drag-and-drop operation.

Favorites and Pins

Favorites are a full-window list for files you want to return to later. Use the Favorites sidebar section to search, open, or remove favorite files. Pins are separate: they appear in Quick Search when the quick search box is empty.

Workspaces

Use Save current workspace in the Workspaces sidebar section to save the current search settings, custom scopes, favorites, Quick Search pins, selected Quick Search indexed roots, and result sort/group/refine state. Selecting a saved workspace restores those settings.

Use the run-on-load button beside a workspace when you want selecting that workspace to immediately run its saved search after the settings are restored. Workspaces without run-on-load enabled restore the setup and wait for you to start the search manually.

Use Import workspace to load a shared workspace JSON file. Use Export workspaces to write all saved workspaces, or the export button beside a workspace to share just that one workspace.

Exporting

Use Export to write the visible results to CSV, JSON, JSON Lines, or Markdown. Active filters and facets are respected, so export after narrowing the list when you only need a subset.

Grounded Answer Drafts

Use Answer to copy an extractive Markdown draft from the visible results. The draft includes cited snippets and source locations only; it does not infer beyond the search evidence. Apply filters and facets first when you want the draft grounded in a narrower result set.

Quick Search Results

Quick Search uses compact result cards with open, reveal, copy, and pin buttons. Keyboard shortcuts are available for the selected row, and dragging a result starts a Windows file drag-and-drop operation.
