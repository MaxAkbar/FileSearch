Search Expressions

Use one query for text and file filters.

Unified mode is the default for the full search window. It combines normal text search with fields such as type:, name:, modified:, and size:.

AI is optional. FileSearch can interpret simple natural-language queries with built-in rules, and semantic: records semantic intent as an explained chip. Real semantic matching requires a future local semantic index; FileSearch does not call paid embedding services.

Modes

Mode Behavior Use when
Unified Parses ordinary content terms plus structured fields. Content terms support AND, OR, NOT, parentheses, quoted phrases, wildcard terms, fuzzy terms, regex terms, and NEAR/n proximity. You want one query like invoice acme modified:last-year type:pdf.
Boolean Parses terms, quoted phrases, slash-delimited regex terms, AND, OR, NOT, and parentheses. You want combinations like (invoice OR receipt) AND NOT draft.
Plain text Treats the whole input as one literal substring. You want an exact phrase with no operators.
Regex Treats the whole input as a .NET regular expression. You need pattern matching such as TODO|FIXME.

Unified Fields

Fields narrow the files that can match. Unfielded words still search file content. In Unified mode, FileSearch shows interpreted chips below the search box so you can see the content terms and filters it understood; edit a chip value to rewrite that token, or remove a chip to remove it from the query.

Field Examples Meaning
name: name:report Only files whose filename contains the value.
path:, folder:, root: path:finance, folder:src Only files under matching paths, folders, or search roots.
content:, regex: content:"termination clause", regex:"TODO|FIXME" Search file contents for a phrase or .NET regular expression.
semantic: semantic:"authentication migration" Records semantic intent as a disabled explained chip until a local semantic index is configured. FileSearch does not call paid embedding services.
ext:, type: ext:pdf, type:excel, type:image Only files with matching extensions or broad file categories.
modified:, created: modified:last-month, created:2026, modified:2026-01-01..2026-03-31 Only files in the requested date range.
size: size:>5mb, size:1mb..10mb Only files within the requested size range.
status:, extractor: status:ok, extractor:pdf Use indexed metadata to filter by index status or extractor id.

Unified Examples

Expression Meaning
invoice acme modified:last-year type:pdf Find PDF content containing both words, limited to files modified in the last year.
PDF invoices from Acme modified last summer Interprets plain language into chips such as Type: pdf, Content: invoice, Content: Acme, and Modified: last-summer.
PDFs about authentication migration modified last year Interprets about authentication migration as a semantic chip and shows it disabled until semantic indexing is available.
name:report path:finance size:>5mb Find matching files by filename/path/size without requiring a content term.
(error OR timeout) AND NOT test path:src Search content under matching source paths while excluding lines containing test.
content:"termination clause" NEAR/25 notice Find lines where the phrase and word appear within 25 word tokens of each other.
invoice~, authentication~2 Use fuzzy content matching. A bare ~ allows one edit; ~2 allows two edits.

Boolean Syntax

Boolean mode supports bare words, quoted phrases, regex terms wrapped in slashes, parentheses, and the operators AND, OR, and NOT. Operators are case-insensitive.

Expression Meaning
error timeout Find files containing both words. Adjacent terms are treated as AND.
error OR warning Find files containing either term.
timeout AND NOT test Find files containing timeout but not test.
"connection string" Find an exact phrase.
(invoice OR receipt) AND NOT draft Group terms to control operator precedence.
/TODO|FIXME/ Use a regex term inside a Boolean expression.

Regex Notes

  • Regex mode uses .NET regular expression syntax.
  • In Boolean mode, a regex term starts and ends with /. Escape a literal slash as \/.
  • Turn on Match case when capitalization must match exactly.

Invalid Queries

If a query cannot be parsed, FileSearch does not start scanning. The status bar shows the parser or regex error so you can fix the expression before trying again.
