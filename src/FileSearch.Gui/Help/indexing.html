Indexed Search

Speed up repeat searches with a background index.

FileSearch can index folders you search often. Indexed searches read prepared text from a local database instead of re-opening every file, and a background watcher keeps the index current while the app is running.

How It Works

  • You choose folders to index, called indexed locations. FileSearch scans them in the background and stores their text in a local database.
  • While the app is open, a file watcher batches changes inside indexed locations and refreshes the affected entries automatically.
  • A search uses the index only when it is covered: the Look in folder falls inside an indexed location and the current options match what was indexed. When a search is not covered, FileSearch automatically falls back to a normal live scan.
Indexed search is opt-in. Turn on the Use index checkbox next to the search controls, or Use index for covered searches in the Manage indexed locations window. Live scan always remains the fallback, so enabling it never makes results worse.

Managing Indexed Locations

Open Index > Manage indexed locations..., or choose Manage locations in the sidebar's Indexed locations section.

Action What it does
Add current folder to index Adds the folder currently shown in Look in as an indexed location.
Add folder... Picks any folder to index.
Rebuild selected Re-scans the selected location from scratch. Use it after changing extraction options or if results look stale.
Remove selected Removes the location and deletes its data from the index database.
Open database Reveals the index database file in Explorer. Also available as Index > Open index location.
Pause / Resume Pauses or resumes all background indexing. Also available from the Index menu.
Indexer resource use Use File > Settings to choose how aggressively background indexing uses CPU and disk. Lower settings pause more often between files.

Location Details

Selecting a location shows its current state:

  • Status, Files, and Lines show how much of the location is indexed.
  • Refresh shows when the location was last indexed.
  • Coverage shows whether subfolders are included or only the top folder.
  • Watcher shows whether file changes are applied automatically.
  • Indexed with records the extraction options used, such as whether Office/PDF documents were included and whether unknown file types were skipped.

Status Badges

Badge Meaning
Ready The location is indexed and has no pending work.
Queued Changes are waiting for the background worker. A count appears when several batches are pending.
Indexing now The location is being scanned.
Paused Background indexing is paused; pending work resumes when you choose Resume.
Watching changes / Watch disabled Whether the file watcher keeps this location updated automatically.
The right side of the status bar shows live background indexing activity, so you can keep searching while the index builds.

Where the Index Is Stored

The index is a local database under %LocalAppData%\FileSearch\Index. It contains text only from the folders you chose to index, never leaves your machine, and can be rebuilt at any time. Removing a location deletes its indexed data.
